# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제:게임 아이템 관리 및 추천 시스템

예) 기분과 시간에 따른 활동 추천기

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

예) 사용자가 현재 기분과 사용할 수 있는 시간을 입력하면, 조건에 맞는 활동을 추천해 주고 싶다.

내 설명:게임에는 다양한 종류의 아이템이 존재한다. 사용자는 원하는 종류의 아이템이나 필요한 능력치를 가진 아이템을 빠르게 찾고 싶지만 아이템이 많아지면 찾기 어렵다. 따라서 사용자가 원하는 조건을 입력하면 조건에 맞는 아이템을 검색하고 추천해 주는 프로그램을 만들고자 한다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 찾고 싶은 아이템 종류(무기, 방어구, 소비 아이템 등)
- 원하는 최소 능력치
- 게임 아이템 데이터

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 2차원 리스트에 저장된 모든 아이템을 반복문으로 검사한다.
- 사용자가 입력한 종류와 일치하는 아이템을 찾는다.
- 능력치가 조건 이상인 아이템만 선택한다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 조건에 맞는 아이템 목록
- 가장 능력치가 높은 추천 아이템

## 6. 사용할 제어구조

### 조건문을 사용할 부분

예) 사용자의 예산보다 가격이 낮은 항목만 추천한다.

내 계획:사용자가 입력한 아이템 종류와 일치하는지 확인하고, 능력치가 기준 이상인 경우에만 결과에 포함한다.


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획:2차원 리스트에 저장된 모든 아이템을 하나씩 검사하여 조건에 맞는 아이템을 찾는다.
