# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제:게임 아이템 관리 및 추천 시스템

이 프로그램을 만든 이유:게임에는 다양한 종류의 아이템이 존재한다. 사용자가 원하는 종류와 능력치를 입력하면 조건에 맞는 아이템을 쉽게 찾을 수 있도록 하기 위해 만들었다. 또한 파이썬의 2차원 리스트, 함수, 조건문, 반복문을 활용하여 데이터를 관리하는 방법을 배우고자 하였다.


---

## 2. 문제 분해 및 모델링

### 입력
아이템 종류(무기, 방어구, 소비)
최소 능력치

### 처리
2차원 리스트에 저장된 모든 아이템을 반복문으로 검사한다.
사용자가 입력한 종류와 같은 아이템을 찾는다.
최소 능력치 이상인 아이템만 결과에 저장한다.
검색된 아이템 중 능력치가 가장 높은 아이템을 찾는다.

### 출력
조건에 맞는 아이템 목록
추천 아이템
최고 능력치

---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:items

한 행의 의미:게임 아이템 1개의 정보

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | 아이템 이름 | 철검 |
| 1번 열 | 아이템 종류 | 무기 |
| 2번 열 | 능력치 | 25 |
| 3번 열 | 가격 | 300 |

이 2차원 리스트를 사용한 이유:여러 개의 아이템 정보를 한 번에 저장하고 관리하기 위해 사용하였다. 각 아이템의 이름, 종류, 능력치, 가격을 하나의 행으로 묶어서 관리할 수 있다.


---

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| get_user_input() | 사용자 입력 받기 | 없음 | 종류, 최소 능력치 |
| search_items() | 조건에 맞는 아이템 검색 | 종류, 최소 능력치 | 검색 결과 리스트 |
| print_result() | 결과 출력 및 추천 | 검색 결과 리스트 | 결과 출력 |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름:search_items()

설명:사용자가 입력한 아이템 종류와 최소 능력치를 기준으로 조건에 맞는 아이템을 찾는 함수이다. 반복문을 사용하여 모든 아이템을 검사하고, 조건문을 이용하여 종류가 같고 능력치가 기준 이상인 경우 결과 리스트에 저장한다. 프로그램의 핵심 검색 기능을 담당한다.


---

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?
아이템의 종류가 사용자가 입력한 종류와 같은지 검사하였다. 또한 아이템의 능력치가 사용자가 입력한 최소 능력치 이상인지 검사하였다.

### 반복문 사용 부분

무엇을 반복 처리했는가?
2차원 리스트에 저장된 모든 아이템을 하나씩 검사하였다.

### 조건문과 반복문이 함께 작동한 부분
반복문으로 모든 아이템을 확인하면서 조건문으로 종류와 능력치를 검사하여 조건에 맞는 아이템만 결과 리스트에 저장하였다.

## 6. 실행 결과

대표 실행 결과를 붙여넣으세요.

```text
아이템 종류: 무기, 방어구, 소비
찾고 싶은 아이템 종류를 입력하세요: 무기
최소 능력치를 입력하세요: 20

검색 결과
------------------------------
이름: 철검, 종류: 무기, 능력치: 25, 가격: 300
이름: 흡혈 낫, 종류: 무기, 능력치: 30, 가격: 800
이름: 철퇴, 종류: 무기, 능력치: 35, 가격: 800
이름: 단검, 종류: 무기, 능력치: 25, 가격: 800
------------------------------
추천 아이템: 철퇴
최고 능력치: 35
```

결과 분석:사용자가 입력한 조건에 맞는 아이템만 검색되었고, 그중 능력치가 가장 높은 철퇴가 추천되었다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력:
무기
20
예상 결과:
능력치가 20 이상인 무기 목록 출력
실제 결과:
예상한 결과와 동일하게 출력됨

### 테스트 2: 예외 또는 경계 사례

입력:
무기
100
예상 결과:
조건에 맞는 아이템이 없음
실제 결과:
"조건에 맞는 아이템이 없습니다." 출력

### 발견한 오류와 수정한 방법
처음에는 결과가 없을 때 추천 아이템을 출력하려고 하여 오류가 발생할 수 있었다. 이를 해결하기 위해 검색 결과의 길이가 0인지 먼저 검사하도록 수정하였다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분:
프로젝트 주제 선정, 2차원 리스트 설계, 함수 분리 방법, 검색 기능 구현 방법

Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분:
예시 아이템 데이터는 사용하지 않고 직접 새로운 아이템 데이터를 추가하였다.

최종 코드에서 내가 직접 판단하거나 수정한 부분:
아이템 이름, 능력치, 가격을 직접 설정하였다. 또한 게임 아이템 관리 시스템이라는 주제를 선택하여 프로그램을 구성하였다.
