# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제:슬롯머신 게임 프로그램

이 프로그램을 만든 이유:파이썬의 함수, 조건문, 반복문, 2차원 리스트를 활용하여 간단한 슬롯머신 게임을 구현하고 사용자와 상호작용하는 프로그램을 만들기 위해 제작하였다.

---

## 2. 문제 분해 및 모델링

### 입력
베팅 금액

### 처리
슬롯 그림 3개를 무작위로 선택한다.
결과를 비교하여 당첨 여부를 확인한다.
당첨 결과에 따라 보상을 계산한다.
소지금을 갱신한다.

### 출력
현재 소지금
슬롯 결과 그림 3개
당첨 여부 및 획득 금액
게임 종료 메시지
---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:symbols

한 행의 의미:하나의 슬롯 그림 정보를 의미한다.

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | 그림 번호 | 1 |
| 1번 열 | 그림 문자 | 🍒 |
| 2번 열 | 사용 안함 |  |
| 3번 열 | 사용안함 |  |

이 2차원 리스트를 사용한 이유:각 슬롯 그림의 번호와 이모지를 하나의 데이터로 묶어서 관리하기 위해 사용하였다. 새로운 그림을 추가할 때도 쉽게 관리할 수 있다.


---

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
|show_money() | 현재 소지금 출력 | money | 현재 소지금 출력|
|spin_slot() | 슬롯 그림 3개 생성 | 없음 | 그림 3개 리스트|
|calculate_reward() | 당첨 여부 확인 및 보상 계산 | bet, result | 보상 금액|
|print_result() | 슬롯 결과와 당첨 여부 출력 | result, reward | 결과 출력|
|main() | 게임 전체 진행 및 관리 | 없음 | 게임 실행|

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름:calculate_reward

설명:슬롯 결과를 분석하여 당첨 여부를 판단하는 함수이다. 3개의 그림이 모두 같으면 베팅 금액의 10배를 지급하고, 그림 2개가 같으면 베팅 금액의 2배를 지급한다. 모두 다르면 보상을 지급하지 않는다. 게임의 핵심 규칙을 담당하는 함수이다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?
슬롯 그림 3개가 모두 같은지 검사
슬롯 그림 2개가 같은지 검사
베팅 금액이 소지금보다 큰지 검사
베팅 금액이 음수인지 검사
종료를 위해 0을 입력했는지 검사

### 반복문 사용 부분

무엇을 반복 처리했는가?
게임이 끝날 때까지 반복 실행
슬롯 그림 3개를 생성하기 위해 반복 처리

### 조건문과 반복문이 함께 작동한 부분
while 반복문 안에서 사용자의 입력을 계속 받으며, 조건문을 통해 종료 여부와 당첨 여부를 판단하였다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력:
1000
예상 결과:
슬롯 결과가 출력되고 당첨 여부에 따라 보상이 계산된다.
실제 결과:
정상적으로 결과가 출력되고 보상이 계산되었다.

### 테스트 2: 예외 또는 경계 사례

입력:
15000
예상 결과:
소지금보다 많은 금액을 베팅할 수 없다는 메시지가 출력된다.
실제 결과:
"소지금보다 많이 베팅할 수 없습니다." 메시지가 출력되었다.

### 발견한 오류와 수정한 방법
숫자가 아닌 문자를 입력하면 프로그램이 오류로 종료되는 문제가 있었다. try-except 문을 사용하여 잘못된 입력이 들어와도 오류 없이 다시 입력받도록 수정하였다.
